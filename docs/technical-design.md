# SolarPulse 技术与设计说明

## 技术架构

```text
Vue 3 + ECharts
        ↓ Axios
Django REST Framework
        ↓ Django ORM
MySQL / SQLite fallback
```

前端不保存业务事实，Metrics、曲线、异常记录和聊天回复均来自 Django API。规则引擎位于 `backend/monitoring/services.py`，便于测试和后续替换为真实预测或诊断服务。

## 视觉论点

SolarPulse 是一块冷静、可信的能源运营中心：蓝灰色分层表面建立管理后台秩序，预测蓝与实际绿承担主要数据语义，橙色用于警告，红色仅用于严重故障。

固定色彩语义：

- 绿色 `#22C55E`：正常、在线、实际功率；
- 蓝色 `#38BDF8`：预测、信息、导航焦点；
- 橙色 `#F97316`：需要研判的警告；
- 红色 `#EF4444`：严重故障与阻断错误；
- 灰色 `#94A3B8`：次级说明。

## 内容计划

1. 顶部提供电站身份、数据状态与导航。
2. Metrics 单一连续指标带，让运维人员先判断整体状态。
3. 预测曲线是主工作区，AI 诊断作为右侧上下文检查器。
4. 底部并列展示子阵列健康状态和聊天助手，承接资产定位与进一步解释。

## 交互论点

1. ECharts 首次加载以克制的曲线绘制动画建立数据层级。
2. Tooltip 将预测、实际、偏差、GHI 和 PR 聚合为一个研判单位。
3. 请求、错误、空数据和聊天等待状态提供明确反馈，避免静默失败。

## 关键业务边界

- `predicted <= 0`：夜间不计算偏差。
- `GHI < 300 W/m²`：低发电优先视为资源不足，不触发设备故障告警。
- 单个异常点：只记录，不形成异常事件。
- 连续两个点偏差超过 20%：触发异常窗口。
- 异常期间平均 GHI 低：优先云层遮挡或预测误差。
- 异常期间 GHI 高：优先局部遮挡、逆变器、组串、积灰或传感器问题。

## 扩展方向

生产环境可进一步接入 SCADA、气象 API、逆变器日志、IV 曲线、清洗记录、工单系统、Redis 缓存、Celery 定时任务和真实大模型服务。
